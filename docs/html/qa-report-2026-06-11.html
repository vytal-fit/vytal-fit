Findings#

P0 — none. No data loss, no cross-tenant leakage observed, no broken core flow.

P1 (should fix in next phase):

  1. No leads.delete — QA/demo leads can only be parked in a stage, never removed. (API gap; the e2e suite works around it.)
  2. Cache-only mutations look real but don't persist: WOD edit modal (no wods.update), plan activate/deactivate/delete (no plans.update/delete), exercise create/edit (router intentionally read-only). Users will believe they saved. Either add the procedures or disable the controls until backed.
  3. Org-isolation UI re-check not performed this pass (org-switch flow wasn't re-driven after the auth flip). Isolation is heavily enforced and tested at the API layer (396 unit tests incl. per-procedure cross-tenant cases), but a UI-level switch-org spot check should be added to the e2e suite.

P2 (polish):

  1. Class-type delete required one retry (likely the in-use CONFLICT path or a stale cache; transient). Watch for recurrence.
  2. Exercises library needed one retry on first load (cold tRPC round-trip without a loading gate on one path).
  3. Dashboard onboarding checklist ("2/10 passos completos") and several KPI tiles (revenue, churn, occupancy) are still mock — fine for demo, label or wire next.
  4. Settings → Website page remains on localStorage by design (publicSite schema mismatch documented in BACKEND_WIRING.md).
